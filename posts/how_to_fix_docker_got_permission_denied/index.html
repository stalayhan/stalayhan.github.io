How to fix Docker Permission Issue

Got permission denied while trying to connect to the Docker daemon socket

docker images
Got permission denied while trying to connect to the Docker daemon socket at unix:///var/run/docker.sock: Get http://%2Fvar%2Frun%2Fdocker.sock/v1.40/images/json: dial unix /var/run/docker.sock: connect: permission denied

TL;DR: Just run below commands from command prompt and skip the rest.

sudo groupadd docker
sudo usermod -aG sudo ${USER}
sudo usermod -aG docker ${USER}
su -s ${USER}
docker run hello-world

You need to do the following:

To create the docker group and add your user: Create the docker group.

sudo groupadd docker

Add your user to the docker group.

# Assuming your $USER is also member of sudoers group.
sudo usermod -aG docker ${USER}

You would need to loog out and log back in so that your group membership is re-evaluated or type the following command:

su -s ${USER}

Verify that you can run docker commands without sudo.

docker run hello-world

This command downloads a test image and runs it in a container. When the container runs, it prints an informational message and exits. If you initially ran Docker CLI commands using sudo before adding your user to the docker group, you may see the following error, which indicates that your ~/.docker/ directory was created with incorrect permissions due to the sudo commands.

If you face an issue like below, please follow this link.

WARNING: Error loading config file: /home/user/.docker/config.json -
stat /home/user/.docker/config.json: permission denied
