Getting Started With Jekyll

Quick-Start Commands

gem install bundler
gem install jekyll # need to installed ruby2.3-dev package

# If the existing directory isn’t empty, you can pass the --force option with jekyll new . --force.
jekyll new my-awesome-site
cd my-awesome-site
bundle exec jekyll serve # => Now browse to http://localhost:4000

# in project folder
bundle info <gem>
bundle update <gem>
bundle exec jekyll serve # browse to localhost:4000

# another way to system wide upgrade to latest Rubygems, run:
gem update --system

Install gem dependencies in jekyll project;

bundler install

Basic Usage

Jekyll Build

jekyll build
# => The current folder will be generated into ./_site

jekyll build --destination <destination>
# => The current folder will be generated into <destination>

jekyll build --source <source> --destination <destination>
# => The <source> folder will be generated into <destination>

jekyll build --watch
# => The current folder will be generated into ./_site,
#    watched for changes, and regenerated automatically.

Jekyll Serve

jekyll serve
# => A development server will run at http://localhost:4000/
# Auto-regeneration: enabled. Use `--no-watch` to disable.

jekyll serve --livereload
# LiveReload refreshes your browser after a change.

jekyll serve --incremental
# Incremental will perform a partial build in order to reduce regeneration time.

jekyll serve --detach
# => Same as `jekyll serve` but will detach from the current terminal.
#    If you need to kill the server, you can `kill -9 1234` where "1234" is the PID.
#    If you cannot find the PID, then do, `ps aux | grep jekyll` and kill the instance.

jekyll serve --no-watch
# => Same as `jekyll serve` but will not watch for changes.
